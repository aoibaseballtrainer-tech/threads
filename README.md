# Threads Analytics Pro

Threadsのアカウント成長を分析・管理するためのプロフェッショナルなソーシャルメディア分析ツールです。

## 機能

- 定期的なデータ追跡と分析
- マルチユーザー管理
- AI によるインサイト生成
- 投稿スケジューリング
- サーバー同期機能

## セットアップ

### 前提条件

- Node.js がインストールされていること
  - インストールされていない場合: `brew install node` または [Node.js公式サイト](https://nodejs.org/) からダウンロード

### クイックスタート

**方法1: バックグラウンドで起動（常時起動用・推奨）**

サーバーを常時起動した状態にしておきたい場合：

```bash
./start-background.sh
```

このスクリプトは以下を実行します：
- Node.jsの確認
- 依存関係のインストール（初回のみ）
- 開発サーバーをバックグラウンドで起動
- ログを `server.log` に保存

**サーバー管理コマンド：**
```bash
# サーバーの状態を確認
./status-server.sh

# サーバーを停止
./stop-server.sh

# ログをリアルタイムで確認
tail -f server.log
```

**方法2: 通常起動（フォアグラウンド）**

ターミナルで直接起動する場合：

```bash
./start.sh
```

このスクリプトは以下を自動で実行します：
- Node.jsの確認
- 依存関係のインストール（初回のみ）
- 開発サーバーの起動

停止するには `Ctrl+C` を押してください。

**方法3: 手動で起動**

1. 依存関係をインストール:
   ```bash
   npm install
   ```

2. `.env.local` ファイルを作成して Gemini API キーを設定:
   ```bash
   echo "GEMINI_API_KEY=your_api_key_here" > .env.local
   ```

3. 開発サーバーを起動:
   ```bash
   npm run dev
   ```

### アクセスURL

開発サーバー起動後、以下のURLでアクセスできます：

- **ローカルアクセス**: `http://localhost:3000`
- **ネットワークアクセス**: `http://[あなたのIPアドレス]:3000`

サーバーが起動すると、ターミナルに表示されるURLからもアクセス可能です。

## ビルドとデプロイ

### 本番用ビルド

```bash
npm run build
```

ビルド後、`dist` フォルダに本番用ファイルが生成されます。

### デプロイ方法

`dist` フォルダの内容をWebサーバーにアップロードしてください。

**推奨ホスティングサービス：**
- Vercel（推奨・無料）
- Netlify（推奨・無料）
- GitHub Pages
- 自前のWebサーバー（Apache、Nginxなど）

### アプリの更新（デプロイ後の修正）

**はい、修正可能です！** デプロイ後も何度でも更新できます。

**更新の流れ：**

1. **コードを修正**
   ```bash
   # ローカルでコードを編集
   # 例: components/ApiSettings.tsx を修正
   ```

2. **ビルド**
   ```bash
   npm run build
   ```

3. **再デプロイ**
   - Vercel/Netlifyの場合：GitHubにプッシュするだけで自動デプロイ
   - 自前サーバーの場合：`dist` フォルダの内容をサーバーにアップロード

4. **完了！**
   - ユーザーがページをリロードすると、新しいバージョンが表示されます

**注意事項：**
- ビルドするたびに新しいバージョンが生成されます
- ブラウザのキャッシュをクリアすると確実に最新版が表示されます
- 環境変数（`.env.local`）は本番環境でも設定が必要です

## サーバー管理

### バックグラウンド起動（常時起動）

サーバーを常時起動した状態にしておく場合：

```bash
# 起動
./start-background.sh

# 状態確認
./status-server.sh

# 停止
./stop-server.sh

# ログ確認
tail -f server.log
```

### 自動起動（macOS起動時に自動でサーバーを起動）

macOS起動時に自動でサーバーを起動したい場合は、`launchd` を使用できます。

## プロジェクト構造

```
├── components/          # React コンポーネント
├── services/            # サービス層（AI など）
├── types.ts            # TypeScript 型定義
├── constants.ts        # 定数定義
├── App.tsx             # メインアプリケーション
├── index.tsx           # エントリーポイント
├── start.sh            # 通常起動スクリプト
├── start-background.sh # バックグラウンド起動スクリプト
├── stop-server.sh      # サーバー停止スクリプト
└── status-server.sh    # サーバー状態確認スクリプト
```
